Embedded Systems Scheduler Priority Queue:

In this project I implemented an application ploting analog sensor data into a graph.

I'm using the ATmega328P and the AVR-GCC cross compiler.

In this project, I implemented a scheduler. The scheduler is based on a self implemented priority queue. The priority queue runns tasks based on a time defined in seconds. For example if the time is 6, than the task will run at second 6 (the start time is 0).
Because it uses a priority queue, it can efficiently get the next task and run it, if the timing is right.

To test this scheduler, I implemented some tasks. One tasks just lets a LED blink, another one reads and ADC value, one tasks prints the entire current priority queue and another tasks adds new tasks to the priority queue.
By that I could ensure the correctness of the scheduler.
